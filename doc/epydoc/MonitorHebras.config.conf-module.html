crear_configuracion2(procesos='', colores=['#00ff00', '#0000ff', '#ffff00', '#000000', '#ff0000', '#ff00..., log_path='/home/manu/.monitorhebras/monitor.log', t_refresco='0.5', tipo_monit=1, num_tasks=100, num_threads=32, load_work='Without ACS', debug=True, get_tid='224', periodicity=1, incr=10, comandos=[], lang='Espa\xc3\xb1ol')

source code 

Crea un nuevo archivo de configuracion con valores por defecto o bien con lo valores especificados. Tambien crea la carpeta de configuracion del proyecto en la home del usuario en caso de no existir.

Parameters:
  • procesos (list) - Una lista con los nombres de los procesos que se quieren monitorizar.
  • colores (list) - Una lista con los colores para cada uno de los estados de un proceso.
  • log_path (string) - Ruta del archivo de log.
  • t_refresco (float) - Tiempo en el que se van monitorizando los procesos.
  • tipo_monit (int) - Tipo de monitorizacion. Puede tomar los valores 1, si la monitorizacion es leyendo de /proc, o 2, si la monitorizacion es del planificador ACS.
  • num_tasks (int) - Numero de tareas ...
  • num_threads (int) - Numero de hebras...
  • load_work (string) - Metodo de carga de trabajo que utilizara el planificador ACS. Toma un valor de los siguientes:
    • "Without ACS"
    • "Signals"
    • "Mono-socket bloqueante"
    • "Mono-socket no bloqueante"
    • "Multi-socket bloqueante"
    • "Multi-socket no bloqueante"
    • "Memoria Compartida"
  • debug (bool) - Indica si el planificador debe o no mostrar mensajes sobre lo que esta pasando con los comandos.
  • get_tid (int) - Numero de la llamada al sistema con la que obtener el TID de una hebra
  • periodicity (int) - Periodicidad de los informes de las hebras al planificador. Puede ser 1, si va a ser mediante incrementos/decrementos de cargas de trabajo o 2, si sera mediante señales.
  • incr (int) - Numero de incrementos/decrementos de cargas de trabajo para informar.
  • comandos (list) - Lista de objetos procinfo.Comandos, con los comandos a ejecutar para que el planificador ACS los planifique.
  • lang (string) - Lenguaje del programa. Puede ser
    • "Español"
    • "English"
Returns: void

Escribe en el archivo de configuracion los nuevos valores establecidos.

Parameters:
  • procesos (list) - Una lista con los nombres de los procesos que se quieren monitorizar.
  • colores (list) - Una lista con los colores para cada uno de los estados de un proceso.
  • log_path (string) - Ruta del archivo de log
  • t_refresco (float) - Tiempo en el que se van monitorizando los procesos
  • lang (string) - Lenguaje del programa. Puede ser
    • "Español"
    • "English"
  • tmp_rep (float) - Tiempo de visualizacion entre registros en la pantalla de reproduccion
  • comenz_rep (int) - Modo en el que se comienza la reproduccion:
    • siempre por el primer registro (PRIMERO - 0)
    • siempre por el actual (ACTUAL - 1)
    • se pregunta al reproducir (PREGUNTAR - 2)
  • sort (int) - El orden que llevaran las hebras al mostrarse en la lista:
    • SORT['PID']
    • SORT['NOMBRE']
    • SORT['ESTADO']
Returns: void
Raises:
  • ConfigError - Cuando no se encuentra el archivo de configuracion o bien cuando este esta mal formado. En cualquier caso la mejor solucion seria volver a crearlo con los nuevos valores.
